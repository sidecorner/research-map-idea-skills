## 0. なぜ今このレポートを書くか

201607.md の4アイデアは「書くことで読める」という交換強制設計が核心であり、これが継続動機の源泉になっている。しかし同じ仕組みが**悪意あるユーザーには攻撃の入口**になりうる。具体的には:

- GPSで現在地を解錠する → 行動パタームが追跡される
- 匿名でメモを置く → ハラスメント・ストーキング誘発メモが投稿される
- 「直前にここにいた人」へのメモが書ける → ターゲット特定型の嫌がらせが可能
- 近くにいる人とマッチングする → 危険な接触のきっかけになる

これらは設計段階から塞がないと、ローンチ後に修正しても「怖いアプリ」の印象が定着して回収不能になる。

## 2. 解消設計（アイデア別）

### 2-A. PassNote（すれ違いメモ交換）改善版

**問題:** 「この場所のメモを全部読める」設計が、人物特定型ハラスメントの受け皿になる。

**解消策:**

#### ① コンテンツ形式の制約（自由記述を廃止）
```
❌ 旧設計: 自由テキスト（何でも書ける）
✅ 新設計: テンプレート + 短い補足
```

テンプレート例:
- `[おすすめポイント]` ＋ 20文字まで
- `[今日の状況]` ＋ 20文字まで（例: 「空いてる」「混んでる」「限定品あり」）
- `[ちょっとした発見]` ＋ 20文字まで

**効果:** 人物描写・特定個人への言及が構造上できなくなる。自由テキストがない → モデレーション負荷も激減。

#### ② タイムスタンプの曖昧化
```
❌ 旧: 「2時間23分前」（逆算でいつ誰がいたか追跡可能）
✅ 新: 「今日の午前中」「数日前」「1週間以内」の3段階のみ表示
```

#### ③ 投稿者ID を場所ごとにリセット
- ユーザーは「場所A での仮名」と「場所B での仮名」が別になる
- サーバー側でも場所をまたいだユーザー同定は行わない
- 同じユーザーが複数の場所にいたことを突き合わせできない

---

### 2-B. TrailNote（残り香情報交換）改善版

**問題:** 「さっきここにいた人が何を感じたか」= 時間差追跡の仕組みに見える。最もリスクが高いアイデア。

**解消策:**

#### ① 「今いる人へのメモ」禁止 → 「場所へのメモ」に設計変更
```
❌ 旧: 「5分前にここを通った私から、今ここを通るあなたへ」（人 → 人）
✅ 新: 「私がここに来たとき感じたこと」（人 → 場所）
```

メモの受け取り側も「この場所に関するメモを読む」であり、「さっきいた特定の人のメモを読む」にならない設計にする。

#### ② 最低遅延 6時間ルール
- 投稿されたメモは **6時間後から** 他のユーザーが読めるようになる
- 「今ここにいる人へ」という用途を物理的に不可能にする
- リアルタイムストーキングツールにならない

#### ③ 場所ごとの「メモバッファ」: 最低5件たまるまで非公開
- 1人しか来ない場所でメモが公開されると、「この人が来た」が特定されやすい
- 最低5件たまって初めて公開 → 誰のメモか特定できない状態を担保

#### ④ 「人物描写」ワードフィルター
- 性別・外見・服装・年齢に関連するキーワードを自動ブロック
- 例: 「女性」「男性」「子供」「若い」「メガネ」「コート」など

### 2-C. SkillSpot（スキルバーター）改善版

**問題:** 近くにいる人とダイレクト交換する設計 = 危険な接触のきっかけになりうる。

**解消策:**

#### ① 非同期化: リアルタイムチャットを廃止
```
❌ 旧: 同じGPSエリア内でリアルタイムマッチング → テキストチャット
✅ 新: 「知識カード」を場所に置く / 拾う（非同期のみ）
```

具体的には:
- 相手と直接やりとりしない
- 「自分の知識カード」を場所に置く（公開メモ形式）
- 「その場所の知識カード」を誰でも読める（匿名）
- リアルタイムの1対1チャットは提供しない

#### ② スキルカードの内容を情報・知識のみに限定
- 「英語OK」→「英語で説明したこの場所のポイント」を書く（スキルそのものではなく、スキルの成果物を置く）
- 「会いに来てください」形式のカードは投稿不可（場所の情報に関係しないカードを弾く）

#### ③ リアル接触の促進を設計から排除
- 「会いましょう」「DM送って」などの接触誘導ワードをブロック
- SNSアカウント・連絡先の共有を禁止（テキストフィルター）

---

### 2-D. KnowLink（知識クレジット）改善版

**問題:** クレジット制が bot 運用・誤情報詐取の動機になる。

**解消策:**

#### ① 回答のクレジット化を廃止 → 「閲覧権」に変更
```
❌ 旧: 回答する → クレジット獲得 → 質問できる（金銭価値型）
✅ 新: 投稿する → その場所での閲覧権が広がる（情報交換型）
```

PassNote と同じロジック: 書く → 読める（クレジットという"通貨"を介さない）

**効果:** クレジット詐取の動機がなくなる。誤情報を流して稼ぐインセンティブが消える。

#### ② 回答の「現地検証スタンプ」制度
- その場所に GPS で訪問したことが確認された人の回答だけが「現地検証済み」バッジを表示
- 訪問履歴のない人の回答は「未検証」として低位置に表示
- 誤情報を流せても信頼性を低く表示できる

## 3. アーキテクチャ共通の悪用対策

### 3-A. GPS座標の量子化（プライバシー設計の基本）

```
❌ 旧: 小数点6桁の緯度経度をそのまま保存（1m精度）
✅ 新: 小数点2桁に丸める（約1km精度のエリア単位）
```

- ユーザーが「どこにいたか」ではなく「どのエリアにいたか」のみ記録
- サーバー側では1km四方のタイル ID のみ保存
- 行動パタームの精密な追跡が不可能になる

### 3-B. サーバー側での位置履歴を保持しない

| データ | 保存先 | 期限 |
|--------|--------|------|
| 投稿したメモ本文 | サーバー（場所タイルIDと紐づけ） | 90日で削除 |
| ユーザーIDと場所の紐づけ | 保存しない | — |
| 「どのユーザーがどの場所に行ったか」 | 保存しない | — |
| 端末ローカル（どこに行ったか） | 端末のみ | ユーザーが任意削除可 |

### 3-C. レート制限（スパム防止）

- 1時間あたり最大3件の投稿（場所単位）
- 同一場所への連続投稿は24時間に1件まで
- 新規インストール後72時間は1日1件まで（信頼スコア段階的開放）

### 3-D. コンテンツモデレーション設計

1. **自動フィルター（第1層）**: 人物描写ワード・接触誘導ワード・わいせつワードをブロック
2. **コミュニティ報告（第2層）**: 読んだメモを「問題あり」と報告 → 3件でシャドウバン
3. **場所キュレーター制（第3層）**: 展覧会・施設の運営者がその場所のメモを管理可能（B2B機能として有料提供）

## 6. 「安全設計」自体をプロダクトの差別化ポイントに変える

悪用対策は「制限」ではなく**ブランド設計**として打ち出せる。

> **「すれ違う情報は、人を特定しない」**

具体的に:
- ランディングページで「人物特定を設計上できないアプリ」と明記
- テンプレート形式のメモが「かえって場所の情報に集中できる」体験として機能
- 施設側に「安全な来訪者フィードバックツール」として導入提案 → B2B訴求が強化

## 7. 開発フェーズ別のリスク管理

### MVP フェーズ（クローズドβ）
- 招待制にすることで悪意あるユーザーの混入を最小化
- メモの全件レビューが現実的な規模（100件以下）に保てる
- フィルターの精度調整も容易

### 公開フェーズ
- コミュニティ報告機能を初日から有効化
- 場所単位での投稿停止機能（荒れた場所を一時クローズ可能）
- 利用規約に「人物特定メモの禁止」を明示

## まとめ

| 悪用リスク | 解消手段 |
|-----------|---------|
| ストーキング誘発メモ | テンプレート形式化 + 人物描写ワードフィルター |
| リアルタイム追跡（TrailNote） | 6時間遅延 + 5件バッファ公開制 |
| 危険な接触誘発（SkillSpot） | 非同期化・チャット廃止 |
| GPS行動履歴の蓄積 | 座標量子化（1km単位）+ 位置履歴をサーバーに持たない |
| botによるクレジット詐取（KnowLink） | クレジット制廃止・閲覧権モデルへ |
| 誤情報拡散 | 現地検証スタンプ + コミュニティ報告 |
| スパム投稿 | レート制限（1時間3件、同一場所24時間1件） |

**最重要変更点の要約（3点）:**
1. **テンプレート形式化** — 自由テキスト廃止がハラスメント防止の最も効果的な手段
2. **TrailNote の6時間遅延** — 設計コストゼロで最大リスクを根絶
3. **SkillSpot の非同期化** — リスク解消と同時に実装コスト削減が実現
